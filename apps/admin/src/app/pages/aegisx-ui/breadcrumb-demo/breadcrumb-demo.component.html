API Reference

Component Inputs

Property Type Default Description

separator string '/' Character to separate breadcrumb items
separatorIcon string undefined Material icon name for separator (e.g., 'chevron_right'). If provided, overrides separator text

Event Type Description

Best Practices

✅ Do

  • Use breadcrumbs for multi-level navigation hierarchies
  • Keep breadcrumb labels concise and meaningful
  • Make all intermediate levels clickable
  • Position breadcrumbs near the top of the page
  • Use icons sparingly to avoid visual clutter
  • Show the current page as the last item (non-clickable)

❌ Don't

  • Use breadcrumbs for single-level navigation
  • Include very long or technical labels
  • Make the current page (last item) clickable
  • Use breadcrumbs as the primary navigation method
  • Show more than 5-6 levels (consider truncation)
  • Mix different separator styles within the same application
