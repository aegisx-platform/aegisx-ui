Overview

AegisX provides two ways to use dialogs:

auto_awesome

AxDialogService

Pre-built confirm dialogs with consistent styling. Perfect for common actions like delete confirmations.

build

MatDialog (Custom)

Custom dialogs using Material Dialog with global AegisX styles. Use for forms and complex interactions.

Simple confirmations, alerts, short forms

Medium (800px)

Standard forms, detailed information 

panelClass: 'dialog-md'

Large (1000px)

Complex forms, multi-column layouts 

panelClass: 'dialog-lg'

Extra Large (1200px)

Data tables, dashboards, extensive content

Size Selection Guidelines

Use dialog-sm (600px) for:
  • Simple confirmations and alerts
  • Short forms (1-3 fields)
  • Quick actions and notifications
Use dialog-md (800px) for:
  • Standard forms (4-8 fields)
  • User profile editors
  • Content preview dialogs
Use dialog-lg (1000px) for:
  • Complex forms with multiple sections
  • Multi-column layouts
  • Rich text editors
Use dialog-xl (1200px) for:
  • Data tables and grids
  • Dashboard configurations
  • Advanced settings panels
Use dialog-fullscreen for:
  • Mobile-first workflows
  • Multi-step wizards (3+ steps)
  • Complex data entry requiring full focus
  • Touch-optimized interfaces

Dialog Best Practices

When to Use AxDialogService
  • check_circle Confirm/cancel decisions (delete, save, discard changes)
  • check_circle Dangerous actions requiring explicit confirmation
  • check_circle Bulk operations on multiple items
  • check_circle Simple yes/no questions
When to Use Custom Dialogs
  • build Forms requiring user input (create, edit)
  • build Multi-step workflows or wizards
  • build Displaying detailed information or previews
  • build Complex interactions requiring custom UI
Styling Guidelines
  • palette Use global styles from _dialog-shared.scss - no component styles needed
  • palette Apply Tailwind classes directly - they work with Material directives
  • palette Use .form-compact wrapper for proper form field sizing
  • palette Leverage header gradient classes for visual hierarchy
Accessibility Considerations
  • accessibility Always provide clear, descriptive titles
  • accessibility Use disableClose: false to allow ESC key dismissal
  • accessibility Ensure focus trap works correctly for keyboard navigation
  • accessibility For dangerous actions, require explicit confirmation (not just Enter key)
