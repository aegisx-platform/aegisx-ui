Key Concepts

Now that you've seen some practical examples, let's review the key concepts you should understand when working with AegisX Design System.

1. Design Tokens

Design tokens are CSS custom properties that define colors, spacing, typography, and other design values. They provide consistency across your application and make theming easy.

Common design tokens:

  • Colors: --ax-text-heading, --ax-text-primary, --ax-background-subtle
  • Spacing: Use rem units that align with our scale (0.5rem, 1rem, 1.5rem, 2rem)
  • Typography: Inherit from Material Design 3 type scales

Learn more in the Design Tokens Guide.

2. Standalone Components

All AegisX examples use Angular's standalone component API. This approach is simpler and more modern than NgModule-based components.

Benefits:

  • No NgModule needed - Import dependencies directly in component
  • Better tree-shaking - Only bundle what you actually use
  • Easier testing - No module configuration required
  • Simpler mental model - Component dependencies are explicit

3. Typography System

AegisX provides two typography systems designed for different use cases:

System Base Size Line Height Use For
AegisX Typography 16px 1.5 UI components, forms, dashboards
Prose Typography 15px 1.7 Documentation, articles, help pages

For long-form content like this documentation, use .ax-prose or .ax-prose-docs classes. See the Typography Guide for details.

Best Practices

Follow these best practices to build maintainable, accessible, and performant applications with AegisX Design System.

Do's

  • Use design tokens - Always use var(--ax-*) instead of hardcoded values
  • Import only what you need - Take advantage of standalone components
  • Follow Material Design guidelines - Use proper spacing, elevation, and component patterns
  • Use semantic HTML - Proper heading hierarchy (h1, h2, h3), semantic tags
  • Test responsiveness - Check your layouts on mobile, tablet, and desktop

Don'ts

  • Don't hardcode colors - Use design tokens for theming support
  • Don't skip accessibility - Material components handle ARIA, but check custom components
  • Don't mix typography systems - Choose the right system for your content type
  • Don't override Material styles unnecessarily - Work with the framework, not against it
Common Mistake: Hardcoding colors like color: #333 instead of using color: var(--ax-text-primary). This breaks theme switching and makes maintenance harder.
