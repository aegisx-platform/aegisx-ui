Prerequisites

Before you begin, ensure you have the following installed on your development machine. AegisX Design System requires Angular 18 or higher and works best with the latest version of Angular Material.

Package Required Version Purpose
Angular 18.0.0 or higher Core framework
Angular Material 18.0.0 or higher UI component library
Node.js 20.0.0 or higher Runtime environment
npm/pnpm Latest Package manager
Note: We recommend using pnpm as your package manager for better performance and disk space efficiency. All examples in this guide use pnpm, but you can use npm if you prefer.

Step 1: Install Required Packages

First, install AegisX UI library and Angular Material in your Angular project. Open your terminal and run one of the following commands based on your package manager:

Using npm

npm install @aegisx/ui @angular/material@18

Using pnpm (Recommended)

pnpm add @aegisx/ui @angular/material@18

This will install both AegisX UI and Angular Material along with all required peer dependencies. The installation process typically takes 1-2 minutes depending on your internet connection.

Step 2: Configure Angular Project

After installing the packages, you need to configure your Angular project to include the AegisX theme styles. Open your angular.json file and add the following to the styles array: 

{{ '{' }}
  "projects": {{ '{' }}
    "your-app": {{ '{' }}
      "architect": {{ '{' }}
        "build": {{ '{' }}
          "options": {{ '{' }}
            "styles": [
              "node_modules/@angular/material/prebuilt-themes/azure-blue.css",
              "node_modules/@aegisx/ui/styles/aegisx-light.css",
              "src/styles.scss"
            ]
          {{ '}' }}
        {{ '}' }}
      {{ '}' }}
    {{ '}' }}
  {{ '}' }}
{{ '}' }}
Important: Make sure to add the AegisX theme after the Angular Material theme to ensure proper style cascading. The order matters!

Step 3: Enable Animations

Angular Material components require animations to work properly. If you haven't already, enable animations in your application configuration (app.config.ts): 

import {{ '{' }} ApplicationConfig, provideZoneChangeDetection {{ '}' }} from '@angular/core';
import {{ '{' }} provideRouter {{ '}' }} from '@angular/router';
import {{ '{' }} provideAnimationsAsync {{ '}' }} from '@angular/platform-browser/animations/async';

export const appConfig: ApplicationConfig = {{ '{' }}
  providers: [
    provideZoneChangeDetection({{ '{' }} eventCoalescing: true {{ '}' }}),
    provideRouter(routes),
    provideAnimationsAsync(), // ✅ Enable animations
  ],
{{ '}' }};

The provideAnimationsAsync() function enables animations with lazy loading, which improves your application's initial load time.

Step 4: Import Theme Styles

Import AegisX theme styles in your global styles.scss file. This gives you access to all design tokens and utility classes: 

@import '@aegisx/ui/styles/aegisx-light.css';

/* Your custom styles */
:root {{ '{' }}
  /* Override design tokens if needed */
  --ax-brand-default: #your-brand-color;
{{ '}' }}

Available Themes

AegisX provides two pre-built themes out of the box:

  • aegisx-light.css - Light theme with Zinc color palette (recommended for most applications)
  • aegisx-dark.css - Dark theme optimized for low-light environments

You can switch between themes by changing the import, or implement dynamic theme switching using Angular's theming system. See the Design Tokens guide for customization options.

Step 5: Verify Installation

Let's verify everything is working correctly by creating a simple component. Create a new component or update an existing one with the following code: 

import {{ '{' }} Component {{ '}' }} from '@angular/core';
import {{ '{' }} MatButtonModule {{ '}' }} from '@angular/material/button';

Run your development server (ng serve or pnpm dev) and navigate to your component. You should see a Material card with proper AegisX styling applied.

Success! If you see the card with the correct colors and typography, your installation is complete. You're ready to start building with AegisX Design System.

Troubleshooting

If you encounter issues during installation, here are some common problems and solutions:

Styles Not Applied

If the AegisX styles aren't being applied to your components, make sure:

  • The theme CSS is imported after Angular Material theme in angular.json
  • You've restarted your development server after configuration changes
  • Your browser cache is cleared (try hard refresh: Ctrl+Shift+R)

Animation Errors

If you see errors related to animations, verify that provideAnimationsAsync() is added to your app.config.ts providers array.

Package Version Conflicts

Ensure all Angular packages are on version 18 or higher. Run ng version to check your installed versions.

Still having issues? Check our GitHub issues or report a new issue with your error message and environment details.
