What are Design Tokens?

Design tokens เป็น CSS custom properties ที่เก็บค่า design ต่างๆ เช่น สี, spacing, typography, shadows ฯลฯ เพื่อให้สามารถนำไปใช้ซ้ำได้ทั่วทั้งแอปพลิเคชัน และสามารถเปลี่ยนแปลงค่าได้จากจุดเดียว

Best Practice: ใช้ design tokens แทนการ hardcode ค่าโดยตรง เช่น ใช้ var(--ax-brand-default) แทน #1976d2 เพื่อให้รองรับ theme switching และง่ายต่อการ maintain

How to Use Design Tokens

มี 3 วิธีหลักในการใช้ design tokens ใน AegisX Design System:

1. Tailwind CSS Classes (แนะนำ)

วิธีที่ดีที่สุดและง่ายที่สุด มี autocomplete ช่วย และ purge unused styles ได้อัตโนมัติ: 

<!-- Colors -->
<div class="text-brand bg-brand-faint">Brand color text</div>
<div class="text-success-default bg-success-faint">Success message</div>

<!-- Typography -->
<h1 class="text-2xl font-semibold">Heading</h1>
<p class="text-sm text-secondary">Subtext</p>

<!-- Spacing -->
<div class="p-6 gap-4">Content with padding and gap</div>

2. CSS Variables (สำหรับ inline styles)

ใช้เมื่อต้องการ dynamic styles หรือ inline styling:

<!-- Inline styles -->
<div style="color: var(--ax-text-primary); background: var(--ax-background-subtle)">
  Content here
</div>

<!-- SCSS files -->
.my-component {{ '{' }}
  color: var(--ax-text-heading);
  background: var(--ax-brand-faint);
  padding: var(--ax-spacing-md);
{{ '}' }}

3. Angular Material Theming

Material components ใช้ theme palette อัตโนมัติ ไม่ต้อง config เพิ่มเติม: 

<!-- Material buttons ใช้ theme palette โดยอัตโนมัติ -->
<button mat-raised-button color="primary">Primary</button>
<button mat-raised-button color="accent">Accent</button>
<button mat-raised-button color="warn">Warn</button>
Tip: ใช้ Tailwind classes เป็นหลัก แล้วใช้ CSS variables เฉพาะเมื่อจำเป็น เช่น dynamic styles หรือ complex calculations

Best Practices

แนวทางในการใช้ design tokens ให้มีประสิทธิภาพสูงสุด:

✅ Do's

  • ใช้ Tailwind classes เป็นหลัก - มี autocomplete และ purge ได้
  • ใช้ semantic tokens - ใช้ .text-success-default แทน .text-green-500
  • รักษา consistency - ใช้ spacing scale ที่กำหนดไว้เสมอ
  • ทดสอบ dark mode - ตรวจสอบทุก component ในทั้ง light และ dark theme
  • ใช้ CSS variables สำหรับ dynamic values - เช่น calculated sizes

❌ Don'ts

  • อย่า hardcode สี - ใช้ #1976d2 แทน design tokens
  • อย่าใช้ค่า spacing แบบสุ่ม - ใช้ 17px แทน spacing scale
  • อย่าข้าม semantic meaning - ใช้ .text-error-default สำหรับสิ่งที่ไม่ใช่ error
  • อย่าผสม systems - ผสม Material Design กับ custom values ที่ไม่ตรงกัน
Common Mistake: การ hardcode สีหรือ spacing ทำให้ยากต่อการ maintain และไม่สามารถรองรับ theme switching ได้ ใช้ design tokens เสมอเพื่อความยืดหยุ่นและสอดคล้องกัน

Customization

สามารถ customize design tokens ได้โดยการ override CSS variables ใน :root: 

/* styles.scss */
:root {{ '{' }}
  /* Override brand color */
  --ax-brand-default: #your-brand-color;
  --ax-brand-faint: #your-brand-light-color;

  /* Override spacing */
  --ax-spacing-md: 20px;

  /* Override typography */
  --ax-font-family-base: 'Your Font', sans-serif;
{{ '}' }}

/* Dark mode overrides */
[data-theme='aegisx-dark'] {{ '{' }}
  --ax-brand-default: #your-dark-brand-color;
{{ '}' }}
Warning: การ customize tokens อาจส่งผลต่อความสอดคล้องของ design system ทำเฉพาะเมื่อจำเป็น และทดสอบให้ครบทุก component ทั้งใน light และ dark mode
