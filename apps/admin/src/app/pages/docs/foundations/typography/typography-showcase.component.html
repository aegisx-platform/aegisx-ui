Typography Overview

AegisX มี 2 ระบบ Typography ที่ออกแบบมาเพื่อวัตถุประสงค์ที่แตกต่างกัน แต่ใช้ Design Tokens เดียวกัน (Zinc color palette) เพื่อความสอดคล้อง

System Base Size Line Height Use For Class
AegisX Typography 16px 1.5 UI components, forms, dashboards .h1, .type-body-*
Prose Typography 15px 1.7 Documentation, articles, long-form content .ax-prose, .ax-prose-docs
ทั้ง 2 ระบบใช้ Zinc color palette ที่อ่านง่ายและนุ่มนวล เหมาะกับการอ่านเนื้อหายาวๆ ดีกว่า Gray palette เดิม

AegisX Typography (UI)

ระบบ typography หลักสำหรับ UI components ใช้ Material Design 3 type scale โดยมี base size 16px และ line-height 1.5 เหมาะสำหรับการสร้าง interface ที่ต้องการความกระชับและชัดเจน

เมื่อไหร่ควรใช้ AegisX Typography

  • สร้าง UI Components (Buttons, Cards, Forms)
  • สร้าง Dashboards และ Data Tables
  • ใช้ Navigation, Headers, Sidebars
  • ต้องการ Material Design 3 consistency
  • เนื้อหาสั้นๆ แบบ UI-focused

Heading Scale

Material Design 3 heading scale สำหรับการสร้าง hierarchy:

Element Class Font Size Weight Line Height
Display Large .h1 36px 700 1.2
Display Medium .h2 30px 600 1.3
Display Small .h3 24px 600 1.4
Headline .h4 20px 600 1.5
Title .h5 16px 600 1.5
Body .type-body-md 16px 400 1.5

Prose Typography (Documentation)

ระบบ typography สำหรับ long-form content ใช้ base size 15px และ line-height 1.7 พร้อม max-width 65ch เพื่อความสบายตาในการอ่าน เหมาะสำหรับ documentation, articles, และเนื้อหายาวๆ

เมื่อไหร่ควรใช้ Prose Typography

  • เขียน Documentation หรือ Help Pages
  • สร้าง Blog Articles หรือ News Posts
  • ทำ About Pages หรือ Landing Pages
  • แสดง Terms & Conditions, Privacy Policy
  • เนื้อหายาวที่ต้องการความสบายในการอ่าน

Prose Elements

Prose typography จัดการ styling ให้กับ HTML elements ทั้งหมดโดยอัตโนมัติ:

  • Headings (h1-h6) - สำหรับโครงสร้างเนื้อหา
  • Paragraphs (p) - ข้อความหลักพร้อม spacing ที่เหมาะสม
  • Lists (ul, ol) - รายการแบบมีจุด/ตัวเลข
  • Tables - ตารางข้อมูลพร้อม hover effects
  • Code blocks (pre, code) - แสดง code พร้อม syntax highlighting
  • Blockquotes - คำพูด/ข้อความอ้างอิง
  • Images (img, figure) - รูปภาพพร้อม captions
  • Links (a) - ลิงก์พร้อม hover effects

Prose Classes

มี 2 ประเภท classes - base classes และ width modifier:

Base Classes (เลือก 1 อัน):

Class Max Width Use For
.ax-prose 90ch Long-form content ทั่วไป
.ax-prose-docs 90ch + Material spacing Documentation ที่มี Material components

Width Modifier (ใช้ร่วมกับ base class ถ้าต้องการ full-width):

Modifier Max Width Example Usage
.ax-prose-full none (full width) class="ax-prose-docs ax-prose-full"
เลือกใช้ง่ายๆ: ใช้ class="ax-prose-docs" สำหรับ documentation ทั่วไป (90ch) | ถ้าต้องการ full-width: เพิ่ม ax-prose-full → class="ax-prose-docs ax-prose-full"

Note Boxes

Prose typography มี note boxes สำหรับเน้นข้อมูลสำคัญ มี 5 ประเภท:

Default Note

Note: ข้อมูลทั่วไปที่ต้องการเน้น ใช้สีเทาเป็นพื้นฐาน

Info Note (Blue)

Info: ข้อมูลเพิ่มเติม คำแนะนำ หรือ tips ใช้สีฟ้าเพื่อบอกว่าเป็นข้อมูลที่เป็นประโยชน์

Warning Note (Yellow)

Warning: คำเตือน ข้อควรระวัง หรือสิ่งที่ควรให้ความสนใจ ใช้สีเหลืองเพื่อดึงดูดความสนใจ

Error Note (Red)

Error: ข้อผิดพลาด ข้อห้าม หรือสิ่งที่ไม่ควรทำ ใช้สีแดงเพื่อเน้นความสำคัญสูง

Success Note (Green)

Success: ผลสำเร็จ best practices หรือสิ่งที่แนะนำให้ทำ ใช้สีเขียวเพื่อบอกว่าเป็นทางที่ถูกต้อง

Code Example

Zinc Color Palette

ทั้ง 2 ระบบ typography ใช้ Zinc color palette ที่อ่านง่ายและนุ่มนวล เหมาะกับการอ่านเนื้อหายาวๆ ดีกว่า Gray palette เดิม

Token Color Hex Value Use For
--ax-text-heading Zinc-950 #0a0a0a Headings, titles (dark, high contrast)
--ax-text-primary Zinc-700 #3f3f46 Body text, paragraphs (softer, easy to read)
--ax-text-secondary Zinc-500 #71717a Secondary text, labels, captions
--ax-text-subtle Zinc-400 #a1a1aa Subtle text, placeholders, disabled state
Why Zinc? Zinc palette มีสีที่นุ่มนวลและอ่านง่ายกว่า Gray palette โดยเฉพาะสำหรับเนื้อหายาวๆ ช่วยลดอาการล้าตาและเพิ่มความสบายในการอ่าน

Best Practices

แนวทางในการใช้ typography systems ให้ได้ประสิทธิภาพสูงสุด:

✅ Do's

  • เลือกระบบที่เหมาะสม - ใช้ AegisX สำหรับ UI, Prose สำหรับ documentation
  • ใช้ design tokens - ใช้ var(--ax-text-*) แทนการ hardcode สี
  • รักษา hierarchy - ใช้ h1-h6 ตามลำดับที่ถูกต้อง (h1 มีหนึ่งต่อหน้า)
  • ใช้ semantic HTML - ใช้ p, ul, ol, table แทนการใช้ div ทั้งหมด
  • เคารพ max-width - อย่าให้ prose content กว้างเกิน 75ch

❌ Don'ts

  • อย่าผสมระบบ - อย่าใช้ .ax-prose ภายใน UI components
  • อย่า override fonts - ใช้ Inter และ Noto Sans Thai เป็นหลัก
  • อย่า hardcode colors - ใช้ design tokens เสมอ
  • อย่าข้าม heading levels - ห้ามกระโดดจาก h1 ไป h3 โดยข้าม h2
